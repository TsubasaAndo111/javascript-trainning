## .png 画像や .jpeg 画像と比べた際の SVG のメリット・デメリット

### メリット

- 拡大縮小しても劣化しない

  SVG は線や図形・パスを数式・ベクトルとして記録いるため、どんなサイズに拡大・縮小してもエッジがシャープに保たれる。ラスター形式（PNG／JPEG）では、元のピクセル数以上にサイズを拡大するとぼやけ・ジャギー・ピクセル化が起きやすい。

- ファイルサイズが小さくなることが多い（単純なグラフィックの場合）

  シンプルなアイコン・ロゴ・図表などでは、SVG の方が同じ視覚内容を表すラスター画像（PNG など）よりもデータ量が少なく済むことがある。

- HTMLに直接埋め込むことができる

  SVG は XML テキスト形式であり、CSS や JavaScript を使って色・形・動きを変更したり、インタラクティブなグラフィックにすることが可能。

- SEOとアクセシビリティのメリット

  SVGはXMLベースのテキストファイルであるため、そのコンテンツは検索エンジンによって読み取られる。
  キーワード、説明、リンクをSVGコード内に直接埋め込むことができるため、SEOで若干のメリットが得られる。

  XML内にtitleタグとdescタグを含めることでスクリーンリーダー用の情報を提供できるため、視覚障害のある方へのアクセシビリティの点でメリットがある。

### デメリット

- 写真・細かい色・グラデーションなどの表現が苦手／不向き

  SVG はベクター形式なので、色の変化が非常に多い写真や複雑なテクスチャ、細かい陰影・光沢といったラスター的な描写では、ファイルが非常に複雑・大きくなるか、描写が適切にできなくなる場合がある。

- 複雑な SVG はパフォーマンス／レンダリング負荷の問題が出ることがある

  SVG ファイルが多数のパス・グループ・フィルター・アニメーションを含むと、特に低スペック端末やブラウザ上での描画・再描画時に遅延や負荷が出ることがある。

- ブラウザ／メールクライアント等での互換性・サポートの限定

  現代のほとんどのブラウザでは SVG 対応が良好だが、レガシーなブラウザ（例：旧 Internet Explorer）や一部メールクライアントでは SVG が正しく表示されなかったり、機能が制限されることがある。

- 編集・作成ハードルが高い・扱いに経験が必要な場合がある

  単純なラスター画像（PNG／JPEG）なら一般的な画像編集ソフトで扱いやすい一方、SVG を一から設計・最適化するにはベクター知識や適切なツール、コード編集などが求められる場合がある。

- セキュリティ上の注意点（特に Web での利用時）

  SVG は XML 形式ゆえにスクリプト・イベント・外部リソース参照を含めることができ、悪意ある SVG を扱うとクロスサイトスクリプティング（XSS）などのリスクがある。
